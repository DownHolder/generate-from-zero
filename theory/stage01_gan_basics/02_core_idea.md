# 2. 核心思想

GAN 之所以有效，依赖于三个相互关联的核心思想：**对抗训练范式**、**隐式密度建模**和**博弈论视角**。理解这三点，就抓住了 GAN 的本质。

## 2.1 对抗训练范式

在传统的监督学习中，我们有一个固定的损失函数和一个固定的目标。模型单方面地逼近这个目标。

GAN 打破了这个模式。两个网络互相竞争，各自优化各自的目标，谁也没有固定的"正确答案"。当我们更新 $D$ 时，$D$ 面对的问题是"在给定当前 $G$ 的条件下，如何最好地区分真假"。当我们更新 $G$ 时，$G$ 面对的问题是"在给定当前 $D$ 的条件下，如何产出最能骗过 $D$ 的样本"。

```
        噪声 z                       真实数据 x
          │                             │
          ▼                             │
     ┌───────────┐                      │
     │   G(z)    │                      │
     │ Generator │                      │
     └────┬──────┘                      │
          │                             │
          ▼                             ▼
     ┌──────────────────────────────────────┐
     │           Discriminator D            │
     │            D(·) → [0, 1]             │
     └─────────────────┬────────────────────┘
                       │
                       ▼
                 真(1) 或 假(0)
```

这个框架的优雅之处在于，$D$ 为 $G$ 提供的损失是**动态自适应的**。当 $G$ 变强时，$D$ 必须更努力地找瑕疵，反过来又鞭策 $G$ 继续进步。只要 $D$ 的容量足够，这个循环不会过早终止。

## 2.2 隐式密度建模

大多数生成模型——例如高斯混合模型、自回归模型、或者我们之后会遇到的 VAE——属于**显式密度模型（explicit density models）**。它们写出 $p_{\text{model}}(\boldsymbol{x})$ 的解析形式，然后通过最大似然来优化参数。

GAN 走的是另一条路：**隐式密度建模（implicit density modeling）**。GAN 没有写出 $p(\boldsymbol{x})$ 的公式。它只定义了一个从噪声到数据的映射 $G(\boldsymbol{z})$。通过从这个映射中采样，$G$ 隐式地定义了生成分布 $p_g$。我们永远不知道 $p_g(\boldsymbol{x})$ 的精确数值，但我们可以从中采样。

这种隐式建模有两个优势：

1. **不需要对数据分布做出限制性假设**。许多显式模型必须假定一个特定的分布族（如高斯），当真实数据不符合这一假设时，模型容量被浪费。
2. **采样是高效的**。显式模型在采样前通常需要运行 MCMC 或其他迭代过程。GAN 的采样只需要一次前向传播：$\boldsymbol{x} = G(\boldsymbol{z})$。

代价是我们失去了对似然 $p_{\text{model}}(\boldsymbol{x})$ 的直接访问，这意味着无法直接比较不同模型在测试集上的 log-likelihood——这是 GAN 评估中的已知难题。

## 2.3 博弈论视角

公式 (1.1) 的 minimax 结构不是偶然的。Goodfellow 等人证明了，在 $D$ 的最优点 $D^*(\boldsymbol{x}) = \frac{p_{\text{data}}(\boldsymbol{x})}{p_{\text{data}}(\boldsymbol{x}) + p_g(\boldsymbol{x})}$ 处，原目标函数等价于最小化以下 Jensen-Shannon (JS) 散度：

$$
V(D^*, G) = 2 \cdot \text{JSD}(p_{\text{data}} \| p_g) - \log 4
$$

其中 JS 散度定义为：

$$
\text{JSD}(P \| Q) = \frac{1}{2}\text{KL}(P \|\| M) + \frac{1}{2}\text{KL}(Q \|\| M), \quad M = \frac{P + Q}{2}
$$

这意味着：训练 $G$ 来最小化 minimax 目标，本质上是在最小化 $p_{\text{data}}$ 与 $p_g$ 之间的 JS 散度。

这个视角解释了 GAN 的诸多特性。JS 散度在非重叠分布上等于常数 $\log 2$，梯度为零——这为 GAN 训练的困难性（模式崩溃、梯度消失）提供了理论解释。后续对此的改进——特别是 Stage 04 讨论的 Wasserstein 距离——正是针对 JS 散度的这一缺陷。

## 2.4 模式崩溃

模式崩溃（mode collapse）是 GAN 训练中最著名的失效模式。当 $G$ 发现少数几个（甚至一个）能稳定骗过 $D$ 的样本后，$G$ 就反复输出这些样本，不再探索数据分布的其他部分。

直觉上，这就像伪造者找到了一个 $D$ 分辨不出的假币模板，于是就只造这一种假币。在 JS 散度框架下，这种情况是"理性"的——如果 $p_g$ 只覆盖 $p_{\text{data}}$ 的一个 mode，而 $D$ 尚未学会注意到其他 mode，那么 $G$ 确实不会受到惩罚。

缓解模式崩溃的方法包括：使用非饱和损失（§1.4）、在隐空间中加入噪声、以及后续 stage 中讨论的 Wasserstein 距离和 Gradient Penalty。
