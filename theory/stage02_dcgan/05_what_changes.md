# 5. 改变了什么，留下了什么

在对比式学习的框架中，每一个 stage 的核心问题是：**相对于上一个 stage，什么变了？什么没变？为什么这个变化是重要的？**

本章回顾 Stage 02 相对于 Stage 01 的增量，并预览它如何为后续 stage 铺路。

## 5.1 Stage 02 的增量

| 维度 | Stage 01 | Stage 02 | 变化类别 |
|------|---------|---------|:--:|
| G 架构 | MLP (Linear + LeakyReLU) | ConvTranspose2d + BN + ReLU | **架构替换** |
| D 架构 | MLP (Linear + LeakyReLU) | Conv2d + BN + LeakyReLU | **架构替换** |
| 下采样 | 无（MLP 展平） | 步幅卷积 | **新增机制** |
| 上采样 | Linear + reshape | 转置卷积 | **新增机制** |
| BatchNorm | 无 | G 和 D 中间层 | **新增机制** |
| 激活函数 | LeakyReLU (G 和 D) | ReLU (G) / LeakyReLU (D) | **分化** |
| 权重初始化 | PyTorch 默认 | $\mathcal{N}(0, 0.02)$ | **新增规范** |
| Loss | BCE | BCE | **不变** |
| 更新比 | 1:1 | 1:1 | **不变** |
| 优化器 | Adam(0.0002, 0.5) | Adam(0.0002, 0.5) | **不变** |
| 带标签？ | 否 | 否 | **不变** |
| 来源 | 自包含脚本 | 依赖 `common/` | **工程重构** |

## 5.2 什么是根本性的改变

在所有这些改变中，有一条是最根本的：**网络知道了空间。**

MLP 的每一层都是对输入向量的线性组合加非线性。从数学上讲，MLP 能做的最复杂的事情是一个高维的连续函数拟合。它可以在像素空间中学出数字的形状，但它不知道 $28 \times 28$ 是一个网格——不知道像素 $(3, 5)$ 和 $(3, 6)$ 是邻居，不知道左右翻转一个数字后它应该看起来相似。

卷积改变了这一切。`Conv2d` 和 `ConvTranspose2d` 的操作定义本身就编码了空间拓扑——$3 \times 3$ 的核在 $28 \times 28$ 的网格上滑动，每一步只看到局部邻域。这不是网络"学到"的，是架构"给定"的。这种**架构层面的归纳偏置（inductive bias）**免去了从头学习"什么是像素相邻"的成本。

## 5.3 代价：训练失衡

这种归纳偏置的成本是训练失衡。在 MLP GAN 中，$G$ 和 $D$ 的能力大致均衡，训练平滑。在 DCGAN 中，$D$ 的结构性优势打破了这种均衡——$D$ 更快地学会区分真假，导致 BCE 饱和，$G$ 的梯度变小。

这不是 DCGAN 的错误。这是"更好的架构"必然带来的副产品。DCGAN 论文解决了这个问题的方式是：**使用更大、更复杂的数据集**——在 CelebA 和 LSUN 上，D 的优势被数据的复杂性冲淡了。

但对我们的 MNIST 学习路线来说，这留下了一个待解决的问题。后续的 Stage 04（WGAN-GP）将用完全相同的 DCGAN 架构，但替换损失函数——用不饱和的 Wasserstein 距离替代 BCE。结果会戏剧性地证明：**改变损失函数比改变架构更能解决训练稳定性问题。**

## 5.4 为未来 stage 铺的几块砖

DCGAN 确立的规范在后续 stage 中被持续使用或有意打破：

- **Stage 03 (cGAN)**：条件 GAN 不是在 DCGAN 的卷积架构上构建的——它回到了 MLP 架构，但在 $G$ 和 $D$ 中加入了标签嵌入（Label Embedding）。理解条件是另一种信息注入方式，和卷积的空间信息注入是正交的改进方向。

- **Stage 04 (WGAN-GP)**：直接使用 DCGAN 的卷积架构，但做了两个关键修改——Critic 中**去掉**所有 BatchNorm（因为 GP 需要逐样本的独立梯度），并将 BCE 损失替换为 Wasserstein 距离。这验证了我们关于"架构 vs 损失函数"的假设。

- **Stage 05 (VAE)**：不是 GAN。但在 VAE 的 Encoder 和 Decoder 中使用类似的卷积/转置卷积架构，可以看到相同的"信息变现"模式——Encoder 将图像压缩为隐变量，Decoder 从隐变量重建图像。

总而言之，DCGAN 的最大遗产不是它的五个架构规则本身——虽然它们被广泛遵循——而是它**证明了架构设计对 GAN 训练的重要性不亚于损失函数的选择**。这个命题贯穿了此后所有的 GAN 研究。
